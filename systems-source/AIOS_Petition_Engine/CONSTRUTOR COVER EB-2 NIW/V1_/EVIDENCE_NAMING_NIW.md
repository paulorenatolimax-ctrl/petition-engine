## REGRAS DE NAMING

### Regra 1: Numeração Sequencial Contínua
- Evidence 1, Evidence 2, ..., Evidence N
- Numeração contínua do início ao fim (não reinicia por seção)
- NUNCA pular números
- Se uma evidence é removida, renumerar TUDO

### Regra 2: Sempre em Inglês
- "Evidence" (não "Evidência")
- SEMPRE com "E" maiúsculo
- SEMPRE com espaço: "Evidence 16" (não "Evidence16")
- NUNCA abreviar: ❌ "Ev.", "E.", "Evid."

### Regra 3: Bold no Corpo
- No Evidence Index: normal (não bold)
- No corpo do texto: SEMPRE **bold**
- Formato: "...conforme documentado em (**Evidence 16**)..."

### Regra 4: Título Descritivo
Cada evidence deve ter título que descreva o documento:
```
✅ Evidence 1. Bacharelado em Sistemas de Informação - Universidade Estácio de Sá
✅ Evidence 8. Scrum Foundation Professional Certification (SFPC™) – CertiProf
✅ Evidence 15. Letter of Recommendation from Wellinton Rodrigues from MV INFORMÁTICA

❌ Evidence 1. Diploma
❌ Evidence 8. Certificado
❌ Evidence 15. Carta
```

### Regra 5: Cartas de Recomendação — Formato Padrão
```
Evidence [N]. Letter of Recommendation from [Nome Completo] from [Organização]
```

### Regra 6: Business Plan — Incluir Nome da Empresa
```
Evidence [N]. Business Plan: [Nome da Empresa LLC]
```

## 3-WAY CONSISTENCY CHECK

Para CADA evidence, verificar match em 3 locais:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   EVIDENCE   │     │   CORPO DO  │     │   ARQUIVO   │
│    INDEX     │────▶│    TEXTO    │────▶│   REAL DO   │
│  (início)   │     │  (prongs)   │     │   CLIENTE   │
└─────────────┘     └─────────────┘     └─────────────┘
      ▲                    ▲                    ▲
      │                    │                    │
      ▼                    ▼                    ▼
  Evidence 12.         "...portfolio       arquivo: "Portfolio_
  Portfolio de         técnico             Desenvolvimento_
  Desenvolvimento     (Evidence 12)..."    Tecnologico.pdf"
  Tecnológico
```

**Match obrigatório**:
1. Número da evidence: IDÊNTICO nos 3 locais
2. Descrição: CONSISTENTE (pode variar em completude, mas não contradizer)
3. Se arquivo tem nome diferente do título no índice → documentar mapping

## EVIDÊNCIAS COMPARTILHADAS ENTRE PRONGS

Algumas evidências são usadas em múltiplos prongs:

```
Evidence [BP]. Business Plan → Prong 1 (PE, merit, importance) + 
                                Prong 2 (plano, financials) + 
                                Prong 3 (empregos, impraticabilidade)

Evidence [Résumé]. Résumé → Eligibility (experiência) + 
                             Prong 2 (trajetória progressiva)

Evidence [Diplomas]. Diplomas → Eligibility (classificação) + 
                                 Prong 2 (formação acadêmica)
```

Quando uma evidence é usada em múltiplos contextos:
- No Evidence Index: listar UMA VEZ, sob a categoria principal
- No corpo: referenciar em cada prong onde relevante
- No quadro consolidado (Conclusão): listar TODOS os prongs associados
