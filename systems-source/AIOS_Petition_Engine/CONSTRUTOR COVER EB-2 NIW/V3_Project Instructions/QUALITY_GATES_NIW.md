### 3.16 Integridade XML Pós-Transformação

> **Lição:** Um script de formatação (bold) reverteu a ordem dos `<w:r>` runs, destruindo parágrafos. Múltiplas versões foram necessárias para corrigir.

**ANTES de qualquer script de transformação XML:**
- [ ] Backup completo do document.xml criado automaticamente
- [ ] Texto puro extraído (concatenação de todos `<w:t>`) e salvo como `pre_transform.txt`

**APÓS qualquer script de transformação XML:**
- [ ] Texto puro extraído novamente e salvo como `post_transform.txt`
- [ ] Comparação `diff pre_transform.txt post_transform.txt`:
  - Se QUALQUER parágrafo mudou de conteúdo textual → **FAIL IMEDIATO — reverter backup**
  - Se parágrafos mudaram de ordem → **FAIL IMEDIATO — reverter backup**
  - Apenas mudanças em `<w:rPr>` (formatação) são permitidas
- [ ] Hash MD5 dos primeiros 100 parágrafos antes/depois idêntico

**Regra absoluta:** Scripts de formatação NUNCA devem mover, reordenar, inserir ou remover elementos `<w:r>`. Apenas modificar `<w:rPr>` dentro de runs existentes.
