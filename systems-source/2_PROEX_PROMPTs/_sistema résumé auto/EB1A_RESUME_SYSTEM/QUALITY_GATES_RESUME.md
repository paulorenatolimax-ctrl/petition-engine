## Gate 4 — Validação de Seção (após produzir cada seção)

### 4A — Completude dos Evidence Blocks

| Check | Status |
|-------|--------|
| Cada evidence block tem MINIATURA ou placeholder [THUMBNAIL] | ☐ |
| Cada evidence block tem TÍTULO | ☐ |
| Cada evidence block tem CRITÉRIO RELACIONADO | ☐ |
| Cada evidence block tem INSTITUIÇÃO/ENTIDADE | ☐ |
| Cada evidence block tem TIPO DE EVIDÊNCIA | ☐ |
| Cada evidence block tem DATA | ☐ |
| Cada evidence block tem LOCAL | ☐ |
| Cada evidence block tem URL (se público) | ☐ |
| Cada evidence block tem DESCRIPTION & IMPACT | ☐ |
| Boxes institucionais presentes onde relevante | ☐ |
| Miniaturas com borda #CCCCCC e 160px largura | ☐ |
| Nenhum evidence block sem miniatura NEM placeholder | ☐ |

### 4B — Consistência Numérica

| Check | Status |
|-------|--------|
| CTRL+F: cada número aparece igual em todo o doc | ☐ |
| CTRL+F: cada data aparece igual em todo o doc | ☐ |
| CTRL+F: cada nome de entidade tem grafia idêntica | ☐ |
| Nenhum número contradiz outro dentro da mesma seção | ☐ |
| Conversão de câmbio correta e com taxa explicitada | ☐ |

### 4C — Forbidden Content Check

| Check | Status |
|-------|--------|
| Zero menção a outros clientes (Carlos, Bruno, PROEX) | ☐ |
| Zero argumentação jurídica (8 CFR, Kazarian, preponderance) | ☐ |
| Zero claims absolutas ("100%", "único", "nenhum outro") | ☐ |
| Zero linguagem de marketing ("incrível", "revolucionário") | ☐ |
| Zero texto em azul | ☐ |
| Zero dados sem fonte | ☐ |
