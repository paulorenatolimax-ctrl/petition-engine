## PROCEDIMENTO DE FALHA

Se qualquer gate FALHA:
1. **PARE** — não avance para a próxima fase
2. **DOCUMENTE** — liste todas as falhas encontradas
3. **CORRIJA** — resolva cada falha
4. **REVALIDE** — execute o gate novamente
5. **REGISTRE** — anote a correção feita

**Princípio**: É mais barato corrigir na fase atual do que descobrir depois.
