# BRIEFING — Formatação Visual dos Documentos

## PROBLEMA

Os dois documentos estão com 80K+ chars de texto corrido puro. Sem tabelas, sem quadros, sem bullet points estratégicos. Visualmente são intragáveis. Um adjudicador do USCIS que recebe centenas de petições precisa conseguir escanear rapidamente.

## REGRA

Adicionar elementos visuais SEM inflar o texto. Isso é reformatação, não expansão. O conteúdo substantivo não muda — a apresentação melhora. Os caracteres totais devem ficar na mesma faixa (80-85K cada).

## REGRAS ABSOLUTAS (CONTINUAM VALENDO)
- ZERO placeholders `[X]` ou brackets
- ZERO "Exhibit" seguido de número
- ZERO auto-auditorias
- ZERO conteúdo em português
- Métricas: máximo 3 ocorrências de cada

## REGRA DE FORMATAÇÃO

- Tabelas em Markdown padrão (`| col1 | col2 |`)
- Bullet points com `-` 
- Bold com `**texto**`
- NÃO usar emojis
- NÃO usar caixas decorativas ou HTML
- Manter parágrafos de prosa entre os elementos visuais — as tabelas COMPLEMENTAM o texto, não substituem
- Os elementos visuais devem facilitar o scanning rápido pelo adjudicador
